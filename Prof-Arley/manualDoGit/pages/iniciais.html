Comandos de inicialização do Projeto

  • git init

    • Iniciamos o controle do git dentro da pagina ( Fazendo o monitoramento da pagina ), toda e qualquer alteração a partir de agora ficara registrado.

  • git branch

    • São versões do desenvolvimento, onde podemos acompanhar cada feito do projeto.

  • git config --global user.name “Pedro”

    • Server Para configurar e registrar o nome do usuário dentro do git.

  • git config --global user.email “joaopedro863@outlook.com”

    • Assim registramos um e-mail no git.

    Podemos usar ( - -local ), que serve para cada repositório. Já o ( - -global ) é para todos os repositórios, temos também o ( Sistem ), serve para todos usuários que locarem na maquina.

  • git config --global init.defautbranch

    • Definimos qual será a raiz do nosso repositório, main ou master.

Comandos para inicialização de acompanhamento

  • git status

    • Mostra quais alterações foram feitas no repositório.

  • git add + “o nome do arquivo”

    • Se criarmos um arquivo ou pasta e queremos que o git fique de olho (registre) as alterações, acompanhe o arquivo, usamos esse comando.

  • git add .

    • Usamos para rastrear tudo oque está no repositório.

  • git commit

    • Comita tudo oque foi feito usando um editor passando a mensagem (Quase nunca usar)

  • git log

    • Mostra quem alterou, quando alterou e oque alterou do repositório e aparece o commit feito.

  • git commit -m “Mensagem que irá aparecer para quem ver oque foi alterado (comitado)”

    • Permite passar uma mensagem com informações de alteração detalhada.

  • git commit -a -m ”Mensagem que irá aparecer para quem ver oque foi alterado (comitado”

    • Adiciona o conteúdo e faz o commit ( Fazendo o commit e o add ).

  • git remote --v

    • Mostra a qual repositório remoto está ligado.

  • git remote add origin https://github/nasicmentold.com

    • Sempre será usado para adicionar o endereço remoto ( Ligar o repositório local, com o repositório remoto. )

O que fazer depois de commitar?

  • Chegamos na hora de jogar as informações realizadas para o GITHUB então segue abaixo os comandos:
  • git push

    • Empurra o arquivo local, para o git remoto.

  • git push origin master

    • Empurra o que está na branch master para o origin que é o remoto.

  • git pull

    • atualizar o local(máquina) com as informações do remoto (hub/github)

  • git pull origin master

    • Puxamos a ultima versão do arquivo remoto, assim atualizamos nosso repositório local.

    obs: origin/master, origin/HEAD (é o que está no github) obs: origin (remoto) head -> master (local)

  • O por que commitar?

    • Ele nada mais é que levar as mudanças de um ambiente local para o repositório no github, permitindo ainda a inserção de uma mensagem descritiva. Assim, a cada mudança ou finalização de uma tarefa, a pessoa desenvolvedora pode submeter seus feitos e deixar claro para as outras pessoas o que ela fez.
